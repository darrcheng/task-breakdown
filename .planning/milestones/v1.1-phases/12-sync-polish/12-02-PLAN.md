   - Call `useSyncStatus()` to get current status
   - Manage `showPopover` boolean state (useState, default false)

     - Icon size: `w-5 h-5` with the mapped color class
     - If `spin` is true, wrap icon in a div with `animate-[spin_2s_linear_infinite]` (slower than default spin, per research recommendation)
     - Button has `relative` positioning for popover anchor
     - On click: toggle `showPopover`

5. **Popover (rendered when showPopover is true):**
   - Position: `absolute right-0 top-full mt-2` (below icon, right-aligned)
   - Style: `bg-white border border-slate-200 rounded-lg shadow-lg p-3 min-w-[200px] max-w-[260px] z-50`
   - Content: The status message text in `text-sm text-slate-700`
   - For error state ONLY: add Retry and Dismiss buttons below the message
     - Retry button: `text-sm font-medium text-blue-600 hover:text-blue-700 px-3 py-1.5 rounded border border-blue-200 hover:bg-blue-50` — calls `retrySync()` then closes popover
     - Dismiss button: `text-sm text-slate-500 hover:text-slate-700 px-3 py-1.5` — just closes popover
     - Buttons in a flex row with gap-2, margin-top mt-2

6. **Outside click handling:**
   - useEffect that adds mousedown listener on document when popover is open
   - If click target is outside the popover ref container, close popover
   - Cleanup removes the listener

7. **Auto-dismiss for non-error states:**
   - useEffect: when popover opens and status is NOT 'error', set a 3-second timeout to close it
   - Clear timeout on unmount or if popover closes early
   - Error popover stays until explicit Retry or Dismiss
