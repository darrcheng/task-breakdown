  <action>
Extend `src/firebase/sync.ts` with the following additions (preserve all existing code):

1. **Add SyncStatus type and state machine at module level:**
   ```typescript
   export type SyncStatus = 'synced' | 'syncing' | 'offline' | 'error';
   let syncStatus: SyncStatus = 'synced';
   const listeners = new Set<() => void>();
   let retryCount = 0;
   const MAX_SILENT_RETRIES = 2;
   const RETRY_DELAYS = [2000, 4000];
   let pendingWrites = 0;
   let syncedDebounceTimer: ReturnType<typeof setTimeout> | null = null;
   ```

2. **Add emitChange helper** that calls all registered listeners.

3. **Add setSyncStatus helper** that updates `syncStatus` and calls `emitChange()`.

4. **Export subscribeSyncStatus(callback)** — adds callback to listeners set, returns unsubscribe function that removes it.

5. **Export getSyncStatusSnapshot()** — returns current `syncStatus` string.

6. **Refactor handleSyncError** from a simple console.error to an async function that:
   - Accepts `(error: unknown, retryFn: () => Promise<void>)`
   - If `retryCount < MAX_SILENT_RETRIES`: waits RETRY_DELAYS[retryCount] ms, increments retryCount, calls retryFn. On success: resets retryCount to 0, calls setSyncStatus('synced'). On failure: recurses.
   - If retryCount >= MAX_SILENT_RETRIES: calls setSyncStatus('error'), resets retryCount to 0.
   - Always console.error the original error.

7. **Export retrySync()** — clears error state by setting status to 'syncing', resets retryCount to 0. The Firestore SDK's `persistentLocalCache` will auto-retry buffered writes; this just clears the UI error state. After a 3-second timeout, if status is still 'syncing', set it to 'synced' (fallback for when there are no pending writes to trigger the normal synced transition).

8. **Add pending write tracking** to the Dexie hook outbound calls:
   - Create helper `trackOutboundWrite(promise: Promise<void>)` that increments pendingWrites, sets status to 'syncing' if not 'offline' or 'error', and on promise settle (then/catch) decrements pendingWrites. When pendingWrites reaches 0 and status is 'syncing', debounce 300ms then set 'synced'.
   - Update the three Dexie hook `.catch(handleSyncError)` calls to instead use `trackOutboundWrite(writePromise.catch(err => handleSyncError(err, retryFn)))` where retryFn re-executes the Firestore write.

9. **Export setupOnlineListener()** — listens for window 'online'/'offline' events:
   - On 'offline': setSyncStatus('offline')
   - On 'online': setSyncStatus('syncing'), then after 3s timeout if still 'syncing' and pendingWrites === 0, set 'synced' (fallback; normally the outbound write completion will set synced)
   - Returns cleanup function

10. **Update startSync()** to call `setupOnlineListener()` and store its cleanup. Also check `navigator.onLine` — if false, set status to 'offline' immediately.

11. **Update stopSync()** to clean up the online listener and reset syncStatus to 'synced', retryCount to 0, pendingWrites to 0.

For the test file, extend `src/firebase/sync.test.ts` with new describe blocks for the sync status state machine. Use `vi.useFakeTimers()` for testing the retry delays and debounce. Mock `navigator.onLine` and window event listeners for online/offline tests. Import the new exports: `subscribeSyncStatus`, `getSyncStatusSnapshot`, `retrySync`, `setupOnlineListener`, and `SyncStatus`.

Note: The handleSyncError signature change means updating the three places in setupDexieHooks where `.catch(handleSyncError)` is called. Each needs to pass a retry function that re-executes the specific Firestore write.
