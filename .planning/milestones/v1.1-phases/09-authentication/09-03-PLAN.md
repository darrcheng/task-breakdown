Present the verification checklist to the user and wait for their test results. This is a human-verify checkpoint -- no code changes.

**What was built:**
Complete authentication system: Google sign-in (popup on desktop, redirect on mobile), auth gate (loading screen -> sign-in screen -> task UI), sign-out with IndexedDB cleanup, and per-user Firestore security rules.

**Verification checklist for user:**

Desktop Browser (Chrome/Firefox):
1. Run `npm run dev` and open `http://localhost:5173`
2. AUTH-04: Verify you see the sign-in screen (centered "taskpad" + Google button), NOT the task UI
3. AUTH-01 + AUTH-05: Click "Sign in with Google" -- a popup should open. Complete Google sign-in. Verify you see the task UI after sign-in.
4. AUTH-02: Refresh the page. Verify you see the loading screen briefly ("taskpad" text), then go directly to the task UI (no sign-in screen). Session persisted.
5. AUTH-03: Open Settings (gear icon). Verify Account section at top shows your avatar, name, and email. Click "Sign out". Verify you return to the sign-in screen immediately.
6. After sign-out, open DevTools > Application > IndexedDB. Verify the TaskBreaker database is gone (clean slate).

Mobile Browser (Safari on iOS / Chrome on Android):
7. Open `http://[your-local-ip]:5173` on your phone (ensure dev server binds to 0.0.0.0: `npm run dev -- --host`)
8. AUTH-05: Tap "Sign in with Google" -- should use redirect (full page navigation to Google), NOT a popup. Complete sign-in. Verify task UI appears.
9. AUTH-03: Open Settings, verify Account section, tap "Sign out". Verify sign-in screen.

iOS Safari Standalone PWA (AUTH-06 -- HARDWARE GATE):
10. On an iOS device, add the app to home screen from Safari
11. Open the home screen app (standalone mode)
12. Tap "Sign in with Google" -- redirect flow should work
13. NOTE: This may fail with the current authDomain (taskpad-app.firebaseapp.com). If it fails, that is expected -- the fix requires deploying to Firebase Hosting (Phase 11) and updating authDomain to taskpad-app.web.app. Document the result either way.

Firestore Rules (DATA-01):
14. Go to Firebase Console > Firestore > Rules Playground
15. Test: Simulate a GET to /users/{your-uid}/tasks/test with authentication as your UID -- should ALLOW
16. Test: Simulate a GET to /users/{different-uid}/tasks/test with authentication as your UID -- should DENY
17. Test: Simulate a GET to /users/{your-uid}/tasks/test without authentication -- should DENY

**Resume signal:** Report results for each numbered item. Type "approved" if all desktop and mobile browser tests pass (iOS standalone can be deferred to Phase 11).
