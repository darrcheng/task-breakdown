  <action>
Replace the deny-all rules from Phase 8 with per-user authenticated rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Per-user data: users/{userId} and all subcollections
    // Phase 10 will write to users/{userId}/tasks/{taskId}, users/{userId}/categories/{catId}, etc.
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    // Deny everything else (no top-level collections accessible)
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

This uses the `users/{userId}/**` wildcard pattern so Phase 10 can create subcollections (tasks, categories) without rule changes.

After writing the file, deploy the rules:
```bash
npx firebase deploy --only firestore:rules
```

Verify deployment succeeded by checking the output for "Deploy complete".
