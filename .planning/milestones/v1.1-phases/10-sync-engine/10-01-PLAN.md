<task type="auto">
  <name>Task 2: Update Firestore config and create sync module contracts</name>
  <files>src/firebase/config.ts, src/firebase/sync.ts, src/firebase/sync.test.ts</files>
  <action>
    1. Update `src/firebase/config.ts`:
       - Replace `import { getFirestore } from 'firebase/firestore'` with:
         `import { initializeFirestore, persistentLocalCache, persistentMultipleTabManager } from 'firebase/firestore'`
       - Replace `export const firestore = getFirestore(app)` with:
         ```
         export const firestore = initializeFirestore(app, {
           localCache: persistentLocalCache({
             tabManager: persistentMultipleTabManager(),
           }),
         });
         ```
       - This enables offline write buffering (SYNC-02) and multi-tab support.

    2. Create `src/firebase/sync.ts` with these exports:
       a. **Module-level state:**
          - `let syncEnabled = false` -- guards Dexie hooks from firing during sign-out
          - `let syncWriteInProgress = false` -- guards against echo loop (inbound writes don't trigger outbound hooks)
          - `let currentUid: string | null = null`
          - `let unsubscribers: (() => void)[] = []` -- onSnapshot unsubscribe functions

       b. **Exported getter functions:**
          - `isSyncEnabled(): boolean` -- returns syncEnabled
          - `isSyncWriteInProgress(): boolean` -- returns syncWriteInProgress

       c. **serializeForFirestore(record, id):**
          - Takes a Dexie record (Task | Category | AISettings) and its numeric ID
          - Returns a plain object with `id` field removed (Firestore doc ID is separate)
          - Keeps all other fields as-is (Firestore SDK auto-converts Date to Timestamp)

       d. **deserializeFromFirestore(data, docId):**
          - Takes Firestore document data and string doc ID
          - Returns object with: `id: Number(docId)`, Timestamp fields converted via `.toDate()`, `parentId` converted to number if present

       e. **startSync(uid: string): void** -- STUB for now
          - Sets `syncEnabled = true`, `currentUid = uid`
          - Logs: "Sync started for user: {uid}" (implementation in Plan 02)

       f. **stopSync(): void** -- STUB for now
          - Sets `syncEnabled = false`, `currentUid = null`
          - Calls each unsubscriber in `unsubscribers` array and clears it
          - Logs: "Sync stopped"

       g. **migrateLocalData(uid: string): Promise<void>** -- STUB for now
          - Logs: "Migration stub called for user: {uid}" (implementation in Plan 03)

       h. **AI settings sync filter:**
          - `const SYNC_BLOCKED_KEYS = ['anthropic-key', 'gemini-key']` (or similar -- check actual key names used in `src/ai/key-storage.ts`)
          - Actually, API keys are stored via WebCrypto in localStorage, NOT in the Dexie aiSettings table. The aiSettings table stores key-value pairs like provider/model preferences. Since API keys never touch Dexie, ALL aiSettings records are safe to sync. No blocklist needed -- just sync the entire aiSettings table.

    3. Update `src/firebase/sync.test.ts`:
       - Update imports to match actual sync.ts exports
       - Make the serialization tests pass (they should now import from the real sync.ts)
       - Run tests to confirm serialization tests pass, placeholder tests remain as todos

    4. Verify the app still builds: `npm run build`
  </action>
