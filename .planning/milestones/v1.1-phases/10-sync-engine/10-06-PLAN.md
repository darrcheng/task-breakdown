<objective>
Fix sign-out destroying Dexie database connection, which causes tasks and categories to disappear after sign-back-in.

Purpose: Both UAT failures (tests 5 and 6) share a single root cause: `db.delete()` in handleSignOut permanently closes the Dexie connection. On re-sign-in, the dead db singleton causes all reads to return empty and all writes to silently fail. Replacing `db.delete()` with per-table `clear()` calls preserves the connection, hooks, and database schema.

Output: Patched SettingsModal.tsx with working sign-out flow, regression test.

```typescript
const handleSignOut = async () => {
  stopSync();

  await signOutUser();
};
```

AFTER:
```typescript
const handleSignOut = async () => {
  stopSync();
  await Promise.all([     // Clear all tables, preserving connection and hooks
    db.tasks.clear(),
    db.categories.clear(),
    db.aiSettings.clear(),
  ]);
  await signOutUser();
};
```

This is the ONLY production change needed. The `db.delete()` import can remain since `db` is still imported for `db.tasks` etc.
