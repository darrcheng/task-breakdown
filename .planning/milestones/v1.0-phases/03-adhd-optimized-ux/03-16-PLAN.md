**Fix B — TaskModal.tsx auto-close on dismiss-triggered create:**

The user wants: when you click out of the modal (backdrop) or press Escape while creating a new task with a title entered, the task should be created AND the modal should close. Currently, `handleSubmit`'s create branch calls `setNavigationOverride(newTask)` to stay in the modal and show BreakdownButton, which makes sense for explicit form submission, but NOT for dismiss-triggered saves.

Add a `closingRef` to distinguish explicit submit from dismiss-triggered submit:

1. Add a ref at the top of the TaskModal component (near other refs):
   ```typescript
   const closingRef = useRef(false);
   ```

2. In `handleBackdropClick` (around line 212), set `closingRef.current = true` BEFORE calling `formRef.current?.submit()`:
   ```typescript
   const handleBackdropClick = () => {
     closingRef.current = true;
     const saved = formRef.current?.submit();
     if (!saved) {
       closingRef.current = false;
       onClose();
     }
     // If saved is true, handleSubmit will check closingRef
   };
   ```

3. In the Escape key handler (around line 82), same pattern — set `closingRef.current = true` before `formRef.current?.submit()`:
   ```typescript
   // Final close — auto-save form data first
   closingRef.current = true;
   const saved = formRef.current?.submit();
   if (!saved) {
     closingRef.current = false;
     onClose();
   }
   ```

4. In `handleSubmit`'s create branch (the `else` block around line 112), after creating the task, check closingRef:
   ```typescript
   } else {
     const newId = await db.tasks.add({
       ...data,
       depth: 0,
       createdAt: new Date(),
       updatedAt: new Date(),
     });
     triggerEstimate(newId as number, data.title, data.description, data.categoryId);

     // If dismiss-triggered (backdrop/Escape), close instead of staying in modal
     if (closingRef.current) {
       closingRef.current = false;
       onClose();
     } else {
       // Explicit submit — stay in modal to show BreakdownButton
       const newTask = await db.tasks.get(newId as number);
       if (newTask) {
         setNavigationOverride(newTask);
       } else {
         onClose();
       }
     }
   }
   ```

This preserves the existing behavior for explicit form submission (Enter or Create button) while adding the new auto-close behavior for dismiss-triggered saves (backdrop click or Escape).
