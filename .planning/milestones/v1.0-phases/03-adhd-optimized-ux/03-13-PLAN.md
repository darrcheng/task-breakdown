**Change 1: TaskForm.tsx — Hide Save/Cancel buttons in edit mode**

Add a new prop `isEditing?: boolean` to TaskFormProps (line 13-27). In the Actions div (line 235), conditionally render:
- If `isEditing` is true: only show the Delete button (left side), hide the Cancel and Save buttons entirely. Keep the sticky positioning.
- If `isEditing` is false (new task mode): show the full actions bar with Cancel + Create button as before.

Implementation — wrap the right-side div (lines 252-266) in a conditional:
```typescript
{!isEditing && (
  <div className="flex items-center gap-2">
    <button type="button" onClick={onCancel} ...>Cancel</button>
    <button type="submit" ...>{submitLabel}</button>
  </div>
)}
```

Also: when isEditing is true AND there's no onDelete, hide the entire actions div to avoid an empty sticky bar.

**Change 2: TaskForm.tsx — Add Enter-in-title to trigger create for new tasks**

Add an onKeyDown handler to the title input (line 131-140) that triggers form submission on Enter ONLY in create mode:
```typescript
onKeyDown={(e) => {
  if (e.key === 'Enter' && !isEditing) {
    e.preventDefault();
    if (title.trim()) {
      onSubmit({ title: title.trim(), description, status, categoryId, date, energyLevel });
    }
  }
}}
```

This lets users type a title and hit Enter to create immediately without needing the Create button.
