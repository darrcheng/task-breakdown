Two changes to fix the Enter-key form submission flow:

**Change 1: CategoryCombobox.tsx — Remove Enter from dropdown-reopen trigger**

In handleKeyDown (line 111-117), the `!isOpen` branch currently opens the dropdown on both ArrowDown and Enter:
```typescript
if (!isOpen) {
  if (e.key === 'ArrowDown' || e.key === 'Enter') {
    setIsOpen(true);
    e.preventDefault();
  }
  return;
}
```

Change this to ONLY open on ArrowDown. Remove Enter from the condition:
```typescript
if (!isOpen) {
  if (e.key === 'ArrowDown') {
    setIsOpen(true);
    e.preventDefault();
  }
  return;
}
```

This way, when the dropdown is closed and Enter is pressed, the event is NOT intercepted — it propagates up to the form, triggering native form submission via the existing submit button.

**Change 2: TaskInlineCreate.tsx — Move Enter-to-submit handler to form level**

Currently handleKeyDown (lines 41-53) is only attached to the title input (line 63). This means Enter from the CategoryCombobox input never reaches it.

Move the handleKeyDown from the title input's onKeyDown to the form's onKeyDown. This catches Enter from ANY input within the form:

1. Remove `onKeyDown={handleKeyDown}` from the title input (line 63)
2. Add `onKeyDown={handleKeyDown}` to the `<form>` element (line 56)
3. Update handleKeyDown to handle Enter generically — since form.requestSubmit() uses inputRef.current?.closest('form'), and we're now on the form itself, simplify to use e.currentTarget (the form) directly:

```typescript
const handleKeyDown = (e: React.KeyboardEvent<HTMLFormElement>) => {
  if (e.key === 'Escape') {
    onClose();
  }
  if (e.key === 'Enter') {
    // Don't intercept Enter when category dropdown is open (it's selecting an item)
    const target = e.target as HTMLElement;
    const combobox = target.closest('[role="combobox"]');
    if (combobox && combobox.getAttribute('aria-expanded') === 'true') {
      return; // Let CategoryCombobox handle Enter for selection
    }
    e.preventDefault();
    if (!title.trim()) return;
    (e.currentTarget as HTMLFormElement).requestSubmit();
  }
};
```

The aria-expanded check ensures that when the dropdown IS open and user presses Enter to select, it still goes to CategoryCombobox's internal handler. Only when the dropdown is closed does Enter submit the form.
