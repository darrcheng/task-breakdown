Complete ADHD-optimized UX layer:

3. Energy tagging — 3-chip selector in TaskForm, badges on cards, energy filter in header

  </what-built>
  <how-to-verify>
Start the dev server: `npm run dev` and open http://localhost:5173

**1. Celebration Animation (TASK-07, ADHD-03)**
- Create a task in list view, cycle it to "done"
- Observe: emerald ring glow appears, text gets strikethrough, row fades out over ~1.5 seconds
- Re-click during fade: should cancel the animation and revert to "todo"
- Open a task with subtasks, complete a subtask — same celebration animation

**2. Start-Here Highlighting (ADHD-05)**
- Open a task that has multiple subtasks (some incomplete)
- Observe: first incomplete subtask has a violet ring border
- Complete that subtask — violet ring should move to the next incomplete subtask
- No text label like "start here" — purely visual

**3. Energy Tagging (ADHD-01)**
- Create a new task — energy chip selector should appear in the form (Low/Medium/High)
- Select "High" energy, save — energy badge (Zap icon) should appear on the task card
- Edit the task — energy level should be preserved
- In the header, click the energy filter chips — only tasks with matching energy level should appear
- Clear filter — all tasks visible again
- Filter applies to both calendar and list views

**4. AI Time Estimates (ADHD-02)**
- Ensure an AI provider is configured (Settings > AI Provider)
- Create a new task with a clear title like "Write a blog post about productivity"
- Wait a few seconds — a time estimate badge should appear on the task card (e.g., "~30m")
- Open the task modal — see estimate detail with override option
- Click to override, enter a different value, confirm
- Override should stick and appear instead of AI estimate
- Create another task in the same category — calibration should influence the estimate

**5. Overdue Banner (ADHD-04)**
- Create a task dated in the past (e.g., 3 days ago) — mark it as "todo"
- Navigate to today in calendar view — amber banner should appear at top
- Banner text should be warm: "You've got X tasks from earlier..." (no guilt language)
- Dismiss the banner — it should not reappear until tomorrow
- Refresh the page — banner should stay dismissed today

**6. Quick Picker (TASK-08)**
- Create 2-3 overdue tasks, open the banner, click "Review"
- Quick picker modal should show all overdue tasks
- Test per-task actions: reschedule (date picker), send to Someday, mark done
- Test "Move all to today" — all tasks should get today's date
- Test "Send all to Someday" — all tasks should move to Someday

**7. Someday View (TASK-08)**
- Switch to Someday view via the view toggle (third icon)
- Verify tasks sent to Someday appear here
- Verify they do NOT appear in calendar or list views
- "Rescue" a task back to a date — it should move out of Someday back to the calendar
- Delete a task from Someday (click-to-confirm pattern)

**Regression Check**
- Drag-and-drop still works in calendar and list views
- Task modal create/edit still works
- AI breakdown (subtask generation) still works
- Keyboard shortcuts all work (j/k/t/m/w/c/l/n/s/?)
