Three surgical fixes in TaskListItem.tsx:

**Fix 1 — Green background during animation (line 39):**
Change `const colors = STATUS_COLORS[task.status]` to `const colors = STATUS_COLORS[displayStatus]`.
This makes the background immediately turn green (bg-emerald-50, border-emerald-300) when displayStatus is set to 'done' at the start of departure, rather than staying yellow/amber because task.status hasn't been written to DB yet.

**Fix 2 — Double-rAF for reliable paint between ring and fade (lines 31-36):**
The current useEffect for departingPhase === 'ring' uses a single requestAnimationFrame. Replace with double-rAF to guarantee the browser paints the ring state before the fade state is applied:

```tsx
useEffect(() => {
  if (departingPhase === 'ring') {
    const rafId = requestAnimationFrame(() => {
      const innerRafId = requestAnimationFrame(() => {
        setDepartingPhase('fade');
      });
      innerRafRef.current = innerRafId;
    });
    return () => {
      cancelAnimationFrame(rafId);
      if (innerRafRef.current !== null) {
        cancelAnimationFrame(innerRafRef.current);
      }
    };
  }
}, [departingPhase]);
```

Add `const innerRafRef = useRef<number | null>(null);` alongside the existing departureTimeout ref.

**Fix 3 — Show-completed: reset departingPhase BEFORE DB write (lines 84-91):**
In the 1500ms timeout callback inside handleStatusClick, reorder operations so `setDepartingPhase(null)` runs BEFORE `await db.tasks.update()`. This prevents the Dexie liveQuery re-render from seeing the component with departingPhase='fade' (opacity-0) when the task is still in the query results (showCompleted=true). The sequence should be:

```tsx
departureTimeout.current = setTimeout(async () => {
  departureTimeout.current = null;
  setDepartingPhase(null);  // Reset FIRST — component shows green done state
  await db.tasks.update(task.id!, {
    status: 'done',
    updatedAt: new Date(),
  });
}, 1500);
```

With Fix 1 + Fix 3 together: when the timeout fires, departingPhase resets to null (removes opacity-0 and ring), and STATUS_COLORS[displayStatus] still shows green done styling because displayStatus was set to 'done' at click time. Then the DB write triggers Dexie re-render which updates task.status to 'done' — but displayStatus was already 'done', so no visual change occurs. Smooth transition.
