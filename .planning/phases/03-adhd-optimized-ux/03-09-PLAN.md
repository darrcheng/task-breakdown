In GeminiProvider.generateSubtasks(), the `config.systemInstruction` is sent unconditionally (line 58). Gemma models (gemma-3-12b-it, gemma-3-27b-it) do not support systemInstruction, causing a 400 error.

**Fix:** Detect Gemma models by checking `this.model.startsWith('gemma-')` and conditionally handle the system prompt:

1. Add a private helper method to the class:
```tsx
private isGemmaModel(): boolean {
  return this.model.startsWith('gemma-');
}
```

2. In generateSubtasks(), conditionally build the API call:
- For Gemma models: prepend the system prompt text to the user message content (e.g., `systemPrompt + '\n\n' + prompt`), do NOT pass `config.systemInstruction`
- For Gemini models: keep existing behavior with `config.systemInstruction`

```tsx
const isGemma = this.isGemmaModel();
const contents = isGemma
  ? [{ role: 'user' as const, parts: [{ text: systemPrompt + '\n\n' + prompt }] }]
  : [{ role: 'user' as const, parts: [{ text: prompt }] }];
const config = isGemma ? {} : { systemInstruction: systemPrompt };

const response = await this.ai.models.generateContentStream({
  model: this.model,
  contents,
  config,
});
```

**Fix 1 — TaskInlineCreate.tsx: Add submit button and Enter handler**

The form has two text inputs (title + CategoryCombobox) and zero submit buttons. Per HTML spec, implicit form submission requires either a submit button or exactly one text input.

Add two things:
1. A visible "Add" submit button after the CategoryCombobox div:
```tsx
<button
  type="submit"
  className="px-3 py-2 text-sm font-medium text-white bg-blue-600 hover:bg-blue-700 rounded-md transition-colors flex-shrink-0"
>
  Add
</button>
```

2. Update the existing handleKeyDown to also handle Enter for explicit submission:
```tsx
const handleKeyDown = (e: React.KeyboardEvent) => {
  if (e.key === 'Escape') {
    onClose();
  }
  if (e.key === 'Enter') {
    e.preventDefault();
    if (!title.trim()) return;
    // Create a synthetic form event to trigger handleSubmit
    const form = inputRef.current?.closest('form');
    if (form) {
      form.requestSubmit();
    }
  }
};
```

Using `form.requestSubmit()` instead of calling handleSubmit directly ensures form validation and the onSubmit handler both fire correctly. The submit button also acts as a visual affordance showing users how to create.

**Fix 2 — TaskForm.tsx: Make action buttons sticky at bottom**

The Actions div (line 235) can be pushed below the fold in modals with maxHeight. Make it sticky so it's always visible:

Change the Actions div className from:
```
"flex items-center justify-between pt-2"
```
to:
```
"flex items-center justify-between pt-3 pb-1 sticky bottom-0 bg-white border-t border-slate-100 -mx-6 px-6 mt-2"
```

The sticky bottom-0 pins it to the bottom of the scrollable modal. The negative margin + padding (-mx-6 px-6) extends the background to the modal edges (matching the parent p-6 padding). The bg-white ensures content scrolling behind is hidden. The border-t adds a subtle separator.

NOTE: The space-y-4 on the parent form will add gap above the sticky bar. The pt-3 replaces the pt-2 for slightly more breathing room above the buttons.
