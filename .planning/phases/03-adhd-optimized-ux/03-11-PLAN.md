Fix TWO bugs in the celebration animation:

**Bug 1: CSS specificity conflict (transition-colors overrides transition-all)**

The root div's clsx (lines 124-132) always has `transition-colors` in the base class (line 125) AND conditionally adds `transition-all` during ring/fade phases. In Tailwind v4, transition-colors appears AFTER transition-all in generated CSS — equal specificity means transition-colors wins, stripping opacity and box-shadow from transition-property.

Fix: Make `transition-colors` conditional — only apply it when NOT departing. Change the base class from:
```
'group w-full flex items-center gap-3 px-4 py-3 rounded-lg border text-left cursor-pointer transition-colors',
```
to:
```
'group w-full flex items-center gap-3 px-4 py-3 rounded-lg border text-left cursor-pointer',
!departing && 'transition-colors',
```

This ensures during ring/fade phases, only `transition-all` is present on the element.

**Bug 2: Ring glow abrupt disappear with show-completed on**

When departingPhase goes from 'fade' to null, ALL ring classes AND transition-all are removed in one render. With show-completed on, the element stays in DOM but ring disappears instantly.

Fix: Add a 'settling' phase to the departingPhase state type. Change state type from `'ring' | 'fade' | null` to `'ring' | 'fade' | 'settling' | null`.

In the 1500ms setTimeout callback (lines 101-111), instead of `setDepartingPhase(null)`:
1. Set `setDepartingPhase('settling')` (this keeps transition-all while removing ring/opacity classes)
2. Use a short setTimeout (400ms) to then set `setDepartingPhase(null)`

In the clsx, add a settling phase class:
```
departingPhase === 'settling' && 'transition-all duration-300',
```
This means during settling: no ring classes (ring fades via transition-all), no opacity-0 (element fades back to full opacity), and transition-all animates the removal smoothly.

Update the `departing` variable to still be true during settling:
```
const departing = departingPhase !== null;
```
This is already correct (departing is true for any non-null phase). But the green text/line-through should NOT show during settling since the task is done and should show normal done styling. Update the departing checks on line-through and text color:
```
departingPhase === 'ring' || departingPhase === 'fade' ? 'line-through decoration-green-600 text-green-600' : null,
```

Also update the re-click cancellation (lines 82-94) to handle 'settling' phase — if user re-clicks during settling, cancel the settling timeout too. Add a settlingTimeout ref alongside departureTimeout.

Clean up: ensure the settling timeout is cleared in the unmount useEffect (lines 70-76).
