**C. Add forwardRef and useImperativeHandle to TaskForm.tsx:**
Convert `TaskForm` to use `React.forwardRef`. Add a ref type:

```typescript
export interface TaskFormHandle {
  submit: () => void;
}
```

Inside the component, use `useImperativeHandle(ref, () => ({ submit: () => { ... } }))`. The `submit` method should:
1. Check if `title.trim()` is non-empty (skip save if empty — user just opened and never typed)
2. Call `onSubmit({ title: title.trim(), description, status, categoryId, date, energyLevel })`

Keep the existing `handleSubmit` for form submission via button/Enter — it calls `e.preventDefault()` and then the same logic.

Export the component as `TaskForm` (same name, just wrapped in forwardRef).

In the backdrop `onClick` (line 191), replace the direct `onClose()` with a function that calls `formRef.current?.submit()` first. The submit itself will trigger `handleSubmit` which calls `onClose()` at the end, so do NOT call `onClose()` separately — the submit flow already includes it.

   However, for the "back to parent" Escape path (when `parentStack.length > 0 || parentTask`), do NOT auto-save — just navigate back as before. Auto-save only applies to the final close.

   For backdrop click: call `formRef.current?.submit()`. If the form has no title (empty), the submit will be a no-op and modal stays open. To handle this edge case: if title is empty, just call `onClose()` directly. The simplest approach: have `submit()` return a boolean indicating whether it actually saved.
