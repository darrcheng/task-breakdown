  <action>
    Run `npm run dev` to start the app locally. Then execute 6 manual test scenarios covering migration, real-time sync, offline-first, conflict resolution, sign-out safety, and category sync. Full test instructions are in the how-to-verify section below.
  </action>
  <what-built>Complete bidirectional sync engine: Dexie hooks push local writes to Firestore, onSnapshot pulls remote changes to Dexie, echo guard prevents loops, LWW resolves conflicts, migration preserves existing data, sign-out is safe.</what-built>
  <how-to-verify>
    Run `npm run dev` to start the app locally.

    **Test 1: First sign-in migration (DATA-02)**
    1. Ensure you have existing local tasks (create a few if needed before signing in)
    2. Sign in with Google
    3. Expect: "Setting up sync..." spinner briefly appears
    4. Expect: All existing tasks are still visible
    5. Open Firebase console > Firestore > users/{your-uid}/tasks -- confirm your tasks are there

    **Test 2: Real-time sync across tabs (SYNC-01)**
    1. Open two browser tabs, both signed in
    2. In Tab 1: create a new task
    3. Expect: Task appears in Tab 2 within ~1 second (no refresh needed)
    4. In Tab 2: edit the task title
    5. Expect: Updated title appears in Tab 1
    6. In Tab 1: delete the task
    7. Expect: Task disappears from Tab 2

    **Test 3: Offline-first (SYNC-02)**
    1. Open DevTools > Network tab > set "Offline" checkbox
    2. Create a new task while offline
    3. Expect: Task appears locally (Dexie write succeeds)
    4. Uncheck "Offline" to go back online
    5. Expect: Task appears in Firebase console (Firestore write happened on reconnect)
    6. Open a second tab -- task should be there too

    **Test 4: Conflict resolution (SYNC-04)**
    1. Open two tabs, both showing the same task
    2. In Tab 1: DevTools > Network > Offline
    3. In Tab 1: edit the task title to "Offline Edit"
    4. In Tab 2 (still online): edit the same task title to "Online Edit"
    5. In Tab 1: go back online
    6. Expect: The task with the LATER edit wins (last-write-wins)

    **Test 5: Sign-out safety**
    1. Confirm you have tasks in Firestore (from prior tests)
    2. Sign out via Settings
    3. Sign back in
    4. Expect: All tasks are restored from Firestore (sign-out did not delete cloud data)
    5. Expect: Local tasks are present again (pulled from Firestore via onSnapshot)

    **Test 6: Category sync**
    1. While signed in, check that default categories (Work, Personal, Health, Learning, Errands) are present
    2. In Firebase console, confirm categories exist under users/{uid}/categories
    3. No duplicate categories should exist
  </how-to-verify>
