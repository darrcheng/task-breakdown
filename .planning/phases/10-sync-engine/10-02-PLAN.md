  <action>
    1. In `src/firebase/sync.ts`, add an exported function `setupDexieHooks()`:
       - This function registers Dexie table hooks on all 3 tables (tasks, categories, aiSettings)
       - It should be called once at app startup (before any writes)

       **For each table (tasks, categories, aiSettings):**

       a. `db.{table}.hook('creating', function(primKey, obj, trans)`:
          - Use `this.onsuccess = (resultKey: number) => { ... }` pattern
          - Inside onsuccess: if `!isSyncEnabled() || isSyncWriteInProgress()` return early
          - Get current uid from module state (`currentUid`)
          - Build doc ref: `doc(firestore, 'users/${currentUid}/{table}/${resultKey}')`
          - Call `setDoc(docRef, serializeForFirestore(obj, resultKey)).catch(handleSyncError)`
          - handleSyncError: `console.error('Sync write failed:', error)` (silent fail, local data safe in Dexie)

       b. `db.{table}.hook('updating', function(modifications, primKey, obj, trans)`:
          - Use `this.onsuccess = () => { ... }` pattern
          - Same guards: `!isSyncEnabled() || isSyncWriteInProgress()` -> return
          - Build doc ref with `primKey`
          - For tasks: use `updateDoc(docRef, serializeModifications(modifications))` -- only send changed fields
          - For categories/aiSettings: use `setDoc(docRef, serializeForFirestore({...obj, ...modifications}, primKey as number), { merge: true })`
          - Catch errors silently

       c. `db.{table}.hook('deleting', function(primKey, obj, trans)`:
          - Use `this.onsuccess = () => { ... }` pattern
          - Same guards
          - Call `deleteDoc(doc(firestore, 'users/${currentUid}/{table}/${primKey}')).catch(handleSyncError)`

    2. In `src/db/database.ts`, add at the bottom (after the db export):
       - Import `setupDexieHooks` from `../firebase/sync`
       - Call `setupDexieHooks()` -- this registers hooks at module load time
       - The hooks are registered once and check `isSyncEnabled()` on each invocation, so they're safe to register even before auth
