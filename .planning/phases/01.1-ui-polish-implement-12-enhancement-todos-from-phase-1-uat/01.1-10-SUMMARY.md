## Decisions Made

- e.preventDefault() added first in every shortcut case — the root cause of 'n' inserting a character was preventDefault not being called before setModalState opened the modal (which auto-focused the title input, then the keyup event fired into it)
- m and w cases also call setViewMode('calendar') so users see the calendar when pressing these from list view
- DatePicker uses only existing date-fns imports already in the project (format, parseISO, addMonths, subMonths) and existing utils/dates.ts functions — no new library dependencies added
- MODAL_HEIGHT increased by 60px (560→620) to accommodate the calendar dropdown (288px wide) without the modal needing to scroll awkwardly
