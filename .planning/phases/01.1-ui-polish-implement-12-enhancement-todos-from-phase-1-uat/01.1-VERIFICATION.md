## Gaps Summary

No gaps found. All 38 must-have truths verified against the actual codebase across all 11 plans. All key links are wired. Build passes with zero errors. No anti-patterns detected.

Plan 11 closed the final 2 gaps from the recheck UAT session:

**Gap 1 — Green checkbox:** `TaskListItem.tsx` previously read checkbox color from `task.status`, which never changes locally during the departure animation (only the `departing` boolean was set). Plan 11 introduces `displayStatus` local state initialized from `task.status` and updated immediately on status click, so the checkbox visually reflects the intended state before the DB write completes 1500ms later.

**Gap 2 — Cancellable departure:** A `departureTimeout` useRef stores the return value of `setTimeout`. The `handleStatusClick` handler now checks `if (departing)` at the top — if true, it clears the ref, resets both `departing` and `displayStatus`, and immediately writes `status: 'todo'` to the DB. A `useEffect` cleanup handles the edge case where the component unmounts mid-departure.

**Gap 3 — CategoryCombobox in inline create:** `TaskInlineCreate.tsx` previously used a native `<select>` element with a `useCategories` hook. Plan 11 removes the hook, the default-category `useEffect`, and the `categories` variable entirely, replacing them with a single `<CategoryCombobox value={categoryId} onChange={setCategoryId} />` in a `flex-[1] min-w-[160px]` wrapper div. The form element has no `overflow-hidden`, so the combobox's `z-50 absolute` dropdown is not clipped.

Commits 7615fcd (green checkbox + cancellable departure) and 893fffb (CategoryCombobox inline create) confirmed in git log.
