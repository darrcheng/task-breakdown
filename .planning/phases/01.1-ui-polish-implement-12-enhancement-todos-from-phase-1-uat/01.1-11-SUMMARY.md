- TaskListItem: re-clicking the checkbox during departure cancels the setTimeout, resets `departing` to false, sets `displayStatus` back to 'todo', and immediately writes todo to DB
- TaskListItem: useEffect cleanup clears the timeout if component unmounts mid-departure
- TaskInlineCreate: native `<select>` removed and replaced with `CategoryCombobox` — inline create now has search, icons, keyboard navigation, and ability to create new categories on the fly

## Files Created/Modified

- `src/components/task/TaskInlineCreate.tsx` - Removed useCategories/categories/default-effect, replaced native select with CategoryCombobox, simplified finalCategoryId fallback

## Decisions Made

- `displayStatus` initialized from `task.status` prop — ensures correct initial color without extra DB reads
- Re-click during departure writes todo to DB immediately (no 1500ms delay) to prevent any race condition where the completion timeout fires after cancellation
- `finalCategoryId = categoryId || 1` fallback in inline create: CategoryCombobox manages its own data so `|| (categories?.[0]?.id ?? 1)` is no longer needed; simple `|| 1` is sufficient
