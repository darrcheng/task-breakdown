**src/ai/providers/types.ts:**
Define the provider interface:
```typescript
export interface SubtaskSuggestion {
  title: string;
  description: string;
}

export interface StreamCallbacks {
  onSubtask: (subtask: SubtaskSuggestion) => void;
  onComplete: () => void;
  onError: (error: Error) => void;
}

export interface AIProvider {
  name: string;
  generateSubtasks(
    taskTitle: string,
    taskDescription: string,
    parentContext: string,
    callbacks: StreamCallbacks,
  ): Promise<void>;
  testConnection(): Promise<boolean>;
}
```

**src/ai/providers/anthropic.ts:**
Implement `AnthropicProvider` using `@anthropic-ai/sdk` with `dangerouslyAllowBrowser: true`. Use `client.messages.stream()` for streaming. Parse response as JSON lines (one subtask per line). Use `content_block_delta` events with `text_delta` type. Buffer text and extract complete JSON lines as subtasks arrive. Call `callbacks.onSubtask()` for each parsed subtask. Handle errors with `callbacks.onError()`. Model: use `claude-sonnet-4-5-20250929` (or latest available).

The streaming parser should:
1. Accumulate text in a buffer
2. Split on newlines
3. Try JSON.parse on each complete line
4. Extract `{ title, description }` from parsed objects
5. Keep incomplete last line in buffer

**src/ai/providers/gemini.ts:**
Implement `GeminiProvider` using `@google/genai`. Use `generateContentStream()` for streaming. Same JSON-lines parsing approach. NOTE: Gemini API may have CORS issues in browser — wrap in try/catch with a clear error message: "Gemini API may require a CORS proxy for browser usage. Consider using Claude provider instead." Model: use `gemini-2.0-flash` (fast, capable).

**src/ai/provider-factory.ts:**
Export `createProvider(name: AIProviderName, apiKey: string): AIProvider` factory function that returns the appropriate provider instance. Throw descriptive error for unknown provider names.

**src/ai/prompts.ts:**
Export `buildSubtaskPrompt(taskTitle: string, taskDescription: string, parentContext: string, count?: number): string` that creates a structured prompt:
- System context: "You are a task breakdown assistant helping users with ADHD break overwhelming tasks into small, actionable steps."
- Instruction: Generate 3-5 concrete subtasks (configurable via count param, default 4)
- Output format: One JSON object per line, each with `title` (string, concise action) and `description` (string, 1-2 sentences of detail)
- Constraints: Make subtasks specific and actionable (not vague), keep titles under 60 chars, include the FIRST step to overcome inertia
- Context: Include parent task title and description for relevance

**src/ai/key-storage.ts:**
Implement encrypted API key storage using Web Crypto API:
- `getOrCreateDeviceKey(): Promise<CryptoKey>` — generates AES-GCM 256-bit key on first use, stores as non-extractable in IndexedDB (separate from Dexie). Retrieves existing key on subsequent calls.
- `saveApiKey(provider: string, apiKey: string): Promise<void>` — encrypts key with device key using AES-GCM with random IV, stores `{ iv, ciphertext }` in localStorage under `taskbreaker-apikey-${provider}`.
- `loadApiKey(provider: string): Promise<string | null>` — decrypts and returns API key, or null if not stored.
- `deleteApiKey(provider: string): Promise<void>` — removes encrypted key from localStorage.

Use a separate IndexedDB database (not Dexie) for the CryptoKey store — name it `TaskBreakerKeyStore`, object store `keys`, key name `device-key`.
