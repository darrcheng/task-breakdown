**src/hooks/useBreakdown.ts:**
Create a hook that orchestrates the entire breakdown lifecycle:

State machine:
```typescript
type BreakdownState =
  | { status: 'idle' }
  | { status: 'configuring' }  // first-use setup needed
  | { status: 'generating'; subtasks: ReviewSubtask[]; progress: number }
  | { status: 'reviewing'; subtasks: ReviewSubtask[] }
  | { status: 'accepting' }
  | { status: 'error'; message: string };

interface ReviewSubtask {
  id: string;           // temp UUID for dnd-kit
  title: string;
  description: string;
  pinned: boolean;
  removed: boolean;
}
```

Hook API:
- `state: BreakdownState` — current state
- `startBreakdown(task: Task)`: Check if provider configured (via useAIProvider.isConfigured). If not, set status to 'configuring'. If yes, set status to 'generating', call `provider.generateSubtasks()` with callbacks:
  - `onSubtask`: append to subtasks array, increment progress
  - `onComplete`: transition to 'reviewing'
  - `onError`: transition to 'error' with message
- `editSubtask(id: string, title: string)`: Update title of subtask in review state
- `removeSubtask(id: string)`: Mark subtask as removed (keep in array for undo potential)
- `reorderSubtasks(activeId: string, overId: string)`: Use arrayMove from @dnd-kit to reorder
- `acceptSubtasks(parentTask: Task)`: Create real tasks in Dexie via `db.tasks.bulkAdd()`:
  - Each subtask gets: parentId = parentTask.id, depth = parentTask.depth + 1, date = parentTask.date, status = 'todo', categoryId = parentTask.categoryId, sortOrder = index
  - Transition to 'idle' after success
- `cancelBreakdown()`: Reset to 'idle'
- `onProviderConfigured()`: Called after ProviderSetupModal completes, re-trigger startBreakdown

Generate unique IDs for ReviewSubtasks using `crypto.randomUUID()`.
