    **src/components/list/DayGroup.tsx:**
    - Renders a date header (e.g., "Sunday, February 22, 2026" or "Today - February 22") followed by TaskListItems for that day
    - Date header is sticky within the scroll container for context while scrolling
    - If no tasks for the day (and showCompleted filters all out), show a subtle "No tasks" placeholder
    - Clickable date header area or "+" button to add task (wired in Plan 04)
    - Props: `date: string`, `tasks: Task[]`, `categoryMap: Map<number, Category>`, `showCompleted: boolean`, `onDayClick: (date: string) => void`, `onTaskClick: (task: Task) => void`

    **src/components/list/ListView.tsx:**
    - Renders days in a vertical scrollable list, starting from 7 days ago to 14 days ahead (21-day initial window)
    - Uses `useTasksByDateRange` to fetch tasks for the visible date range
    - Groups tasks by date and renders a DayGroup for each day
    - **Infinite scroll:** Uses IntersectionObserver on sentinel elements at top and bottom:
      - When bottom sentinel is visible, extend the date range forward by 7 days
      - When top sentinel is visible, extend the date range backward by 7 days
    - **"Today" button:** Floating button (fixed position, bottom-right corner) that scrolls the list to today's DayGroup using `scrollIntoView({ behavior: 'smooth' })`
    - Assign `id="day-{date}"` to each DayGroup wrapper div for scroll targeting
    - Props: `showCompleted: boolean`, `onDayClick: (date: string) => void`, `onTaskClick: (task: Task) => void`
    - Use `useRef` for the scroll container and `useState` for the date range bounds
    - On initial mount, auto-scroll to today

    Performance considerations:
    - Only query tasks for the current visible date range (not all tasks)
    - IntersectionObserver is more efficient than scroll event listeners
    - Each date range extension is small (7 days) to keep queries fast
