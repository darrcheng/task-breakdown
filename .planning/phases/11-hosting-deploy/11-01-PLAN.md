<task type="auto">
  <name>Task 1: Configure Firebase Hosting and deploy scripts</name>
  <files>firebase.json, package.json</files>
  <action>
  1. Update firebase.json to add the `hosting` section alongside the existing `firestore` section. The complete firebase.json should be:

  ```json
  {
    "firestore": {
      "rules": "firestore.rules"
    },
    "hosting": {
      "public": "dist",
      "ignore": [
        "firebase.json",
        "**/.*",
        "**/node_modules/**",
        "src/**",
        ".planning/**",
        ".env*"
      ],
      "rewrites": [
        {
          "source": "**",
          "destination": "/index.html"
        }
      ],
      "headers": [
        {
          "source": "/sw.js",
          "headers": [
            { "key": "Cache-Control", "value": "no-cache" }
          ]
        },
        {
          "source": "/index.html",
          "headers": [
            { "key": "Cache-Control", "value": "no-cache" }
          ]
        },
        {
          "source": "/assets/**",
          "headers": [
            { "key": "Cache-Control", "value": "max-age=31536000, immutable" }
          ]
        },
        {
          "source": "**/*.{ico,png,svg,webmanifest}",
          "headers": [
            { "key": "Cache-Control", "value": "max-age=86400" }
          ]
        }
      ],
      "predeploy": ["npm run build"]
    }
  }
  ```

  Key points:
  - `public: "dist"` — Vite build output directory (not "public" or "build")
  - SPA rewrite: all routes serve index.html (handles /someday refresh)
  - no-cache for sw.js and index.html per user decision
  - immutable for hashed assets in /assets/ per user decision
  - 24-hour cache for static assets (icons, images, manifest) — Claude's discretion
  - predeploy runs `npm run build` automatically — build only, no lint or type-check per user decision
  - `--only hosting` is NOT here — that goes in the npm script (predeploy just builds)

  2. Add two scripts to package.json `scripts` section:
  - `"deploy": "firebase deploy --only hosting"` — the primary deploy command
  - `"deploy:rules": "firebase deploy --only firestore:rules"` — explicit separate rules deploy

  Do NOT modify any other scripts. Do NOT add lint or type-check to the deploy pipeline per user decision.
  </action>
