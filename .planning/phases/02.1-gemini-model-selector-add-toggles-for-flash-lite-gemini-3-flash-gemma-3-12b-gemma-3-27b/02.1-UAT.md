expected: Open Settings. With Gemini selected as provider and a valid API key saved, model toggle buttons appear below the API key section showing 5 model options.

### 2. Five Model Options Displayed
expected: The toggle buttons should show: Flash, Flash Lite, Gemini 3 Flash, Gemma 3 12B, and Gemma 3 27B (or their display names). All 5 are visible.
result: pass

### 3. Active Model Highlighted
expected: One model button (default: Flash) should have a blue highlighted border/style distinguishing it from the others.
result: pass

### 4. Switch Model Instantly
expected: Click a different model button (e.g., Gemma 3 27B). Selection switches immediately — no loading spinner, no API call, no delay. The clicked button becomes highlighted.
result: pass

### 5. Model Persists Across Refresh
expected: Select a non-default model (e.g., Flash Lite), then refresh the page. Open Settings again — the previously selected model should still be highlighted.
result: pass

### 6. Model Selector Hidden During Key Input
expected: When the API key input field is visible (before saving a key, or after clicking to change key), the model selector toggle buttons should not be shown.
result: pass

### 7. Clear Provider Resets Model
expected: Clear the Gemini provider (remove API key). Re-add Gemini with a new key. The model selection should reset to the default (Flash), not retain the previously selected model.
